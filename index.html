Nevron Brand Agent

A Claude Code agent that enforces Nevron brand consistency.
Across web, presentations, documents, and print.

01 — What is this

A brand specialist, on demand

The nevron-brand agent plugs into Claude Code and enforces Nevron's brand system — colors, typography, logos, spacing, gradients — anywhere you're building or reviewing Nevron-branded work.

It knows the full brand spec (colors, typography, gradients, corner radius rules, logo variants, PrimeIcons), has access to every brand asset in this repository, and applies the rules automatically instead of you having to remember them.

Scope: Nevron-branded work only — NevronCore, company site, client pitches, internal docs. Not for non-Nevron projects or unrelated brands.
02 — Quick start

Install in under a minute

Fastest path: use Copy install prompt at the top of this page and paste it into a fresh Claude Code session — Claude handles all of the below. Prefer to do it manually? Follow the steps.

  1. Clone the repo into the cache location

    Full brand repo lives here: agent, BRAND spec, tokens, assets. Claude Code's agent Read/Grep into this folder on demand.

    mkdir -p ~/.claude/mcp-servers
cd ~/.claude/mcp-servers
git clone https://github.com/kajasajn/nevron-brand-agent.git

  2. Install the agent file (symlink on Unix, copy on Windows)

    Claude Code looks for user-installed agents in ~/.claude/agents/. Unix supports symlinking (auto-updates on git pull). Windows without Developer Mode needs a real copy — which means re-running a copy step on every update.

    mkdir -p ~/.claude/agents

# Detect platform
case "$(uname -s)" in
  MINGW*|MSYS*|CYGWIN*)   PLAT=windows ;;
  *)                      PLAT=unix ;;
esac

if [ "$PLAT" = "windows" ]; then
  cp ~/.claude/mcp-servers/nevron-brand-agent/agent/nevron-brand.md ~/.claude/agents/nevron-brand.md
else
  ln -s ~/.claude/mcp-servers/nevron-brand-agent/agent/nevron-brand.md ~/.claude/agents/nevron-brand.md
fi

  3. Set BRAND_REPO to your machine's cache path

    The agent ships with the author's dev path hardcoded. Override it so Discovery never has to run on first use.

    TARGET="$HOME/.claude/mcp-servers/nevron-brand-agent"

if [ "$PLAT" = "windows" ]; then
  sed -i.bak "s|^BRAND_REPO:.*|BRAND_REPO: $TARGET|" ~/.claude/agents/nevron-brand.md
  rm -f ~/.claude/agents/nevron-brand.md.bak
else
  sed -i.bak "s|^BRAND_REPO:.*|BRAND_REPO: $TARGET|" ~/.claude/mcp-servers/nevron-brand-agent/agent/nevron-brand.md
  rm -f ~/.claude/mcp-servers/nevron-brand-agent/agent/nevron-brand.md.bak
  git -C ~/.claude/mcp-servers/nevron-brand-agent update-index --assume-unchanged agent/nevron-brand.md
fi

  4. Restart Claude Code

    Close and reopen Claude Code, or start a new session. The agent loads with the correct tool list (Read, Write, Edit, Grep, Glob).

  5. Verify

    Try asking Claude:

    "Use the nevron-brand agent to return the M3 brand blue hex."

    Expected answer: #1B92FF. If the agent isn't found, double-check the symlink or copy at ~/.claude/agents/nevron-brand.md.

03 — Auto-trigger

Make it call itself

By default you invoke the agent by name. To have Claude call it automatically on any Nevron-branded task, paste this prompt into Claude Code once:

Add the nevron-brand agent to my CLAUDE.md so it auto-triggers for any Nevron brand tasks.

Ask me which projects I want to add it to — show me a multiple selection list of my projects that have a CLAUDE.md file. Also give me the option to add it globally to ~/.claude/CLAUDE.md.

Add it to the agents table (or create one if it doesn't exist) with this row:
| **nevron-brand** | Nevron brand colors, typography, logo usage, spacing, brand guidelines, any Nevron-branded design decisions | User asks about Nevron brand, or is building/reviewing anything Nevron-branded |

Also add this to the agent decision flow (or create one): "Nevron brand question / building Nevron UI? → nevron-brand"

Claude will ask which projects to configure, then handle the rest.

What changes after auto-trigger is set up

  • Ask about Nevron colors / fonts / guidelines → agent called automatically
  • Build any Nevron-branded component → agent called first
  • Review a design for brand consistency → agent consulted
  • Tokens, logos, typography used correctly without you specifying
04 — Updating

Pulling in new brand changes

Fastest path: Copy update prompt at the top of this page and paste into Claude Code — it handles fetch, diff preview, backup, pull, and (on Windows) the re-copy. Prefer manual? Follow these steps.

cd ~/.claude/mcp-servers/nevron-brand-agent

# Preview what's new
git fetch origin
git log --oneline HEAD..origin/master
git diff HEAD..origin/master --name-status | grep -E "^(A|R|D)"   # added / renamed / deleted
git diff HEAD..origin/master -- agent/nevron-brand.md              # agent file changes

# Backup current agent file before pulling
TS=$(date +%Y%m%d-%H%M%S)
cp -L ~/.claude/agents/nevron-brand.md ~/.claude/agents/nevron-brand.md.bak-$TS

# Pull
git pull origin master

# On Windows, re-copy + re-apply BRAND_REPO (symlink users can skip this)
if [ ! -L ~/.claude/agents/nevron-brand.md ]; then
  TARGET="$HOME/.claude/mcp-servers/nevron-brand-agent"
  cp ~/.claude/mcp-servers/nevron-brand-agent/agent/nevron-brand.md ~/.claude/agents/nevron-brand.md
  sed -i.bak "s|^BRAND_REPO:.*|BRAND_REPO: $TARGET|" ~/.claude/agents/nevron-brand.md
  rm -f ~/.claude/agents/nevron-brand.md.bak
fi

Restart Claude Code to reload the agent with the new tools list or instructions.

Why the re-copy step: On Unix the installed agent is a symlink, so git pull updates it automatically. On Windows without Developer Mode it's a regular file, so you have to cp it again after every pull.
